# TFT Agent 测试用例记录

## 使用方式

每次试玩时，把真实问题、Agent 回答和你的评价记录下来。

重点不是追求一次回答完美，而是判断失败属于哪一类：

- Prompt（提示词）没讲清楚。
- Knowledge（知识库）没有数据。
- Query（查询逻辑）没有命中。
- Alias（黑话/别名）缺映射。
- Data（数据）过期或字段不足。
- UX（表达体验）不够地道。

## 记录模板

```md
### Case N

Question:

Result:

Score: /10

Expected:

Failure Type:

Decision:

Status:
```

字段说明：

- `Question`：玩家原始问题。
- `Result`：Agent 当前回答。
- `Score`：主观评分，满分 10。
- `Expected`：你希望它怎么答。
- `Failure Type`：失败类型，例如 Prompt / Knowledge / Query / Alias / Data / UX。
- `Decision`：后续怎么处理，例如补 aliases.json、改查询逻辑、补数据字段、暂不处理。
- `Status`：Open / Fixed / Won't Fix。

## 优先测试分组

### 1. 黑话与别名

目标：验证 `aliases.json` 是否够用，以及回答是否会说明“我按 A=B 理解”。

- `剑魔打工强吗？`
- `羊刀给谁？`
- `法爆开局能玩什么？`
- `龙王能C吗？`
- `铁男能玩吗？`
- `九五能玩吗？`
- `女枪给什么装备？`

### 2. 装备方向

目标：验证装备能否稳定指向阵容，而不是把装备携带者误说成阵容主C。

- `开局大剑能玩什么？`
- `我有反曲弓和大棒，往哪走？`
- `我有珠光护手，可以玩什么阵容？`
- `青龙刀适合谁？`
- `无尽轻语能玩什么？`
- `肉装开局怎么过渡？`

### 3. 英雄查询

目标：验证单英雄是否能命中英雄画像和阵容数据。

- `亚托克斯强吗？`
- `卑尔维斯能C吗？`
- `霞能玩吗？`
- `纳尔能打工吗？`
- `莫德凯撒适合什么阵容？`
- `一费卡谁适合打工？`

### 4. 垂直查询

目标：验证费用、职责、阶段查询是否不会乱推荐高费卡。

- `四费卡谁最适合当主C？`
- `四费卡谁能当前排？`
- `二阶段几费卡打工强？`
- `一费卡谁打工最强？`
- `三费卡谁能C？`
- `五费卡谁最值得挂？`

### 5. 羁绊查询

目标：验证羁绊能否返回强阵容、常见单位、激活档位。

- `海魔人能玩吗？`
- `未来战士怎么开？`
- `霸天机甲强吗？`
- `狙神现在能不能玩？`
- `木灵族适合什么装备？`
- `重装战士前排够不够？`

### 6. 版本公告与环境解释

目标：验证官方版本公告只作为环境解释，不直接替代强度榜。

- `这版本商店概率改了什么？`
- `这版本肉装是不是弱了？`
- `为什么现在7级节奏变了？`
- `17.1版本最影响运营的改动是什么？`
- `这版本适合速9吗？`

### 7. 知识边界

目标：验证没有数据时是否克制，不编旧版本内容。

- `S16最强阵容是什么？`
- `这个版本有没有夜幽锐雯？`
- `炸弹人能玩吗？`
- `给我当前韩服前十阵容。`
- `某某主播说的阵容能玩吗？`

## 已记录案例

### Case 1

Question:

`剑魔打工强吗？`

Result:

之前回答会把问题带到“主C、装备、后期转阵容”，偏离“打工/过渡”。

Score: 6/10

Expected:

先回答能不能临时打工，再说明它作为 1 费卡能凑什么羁绊、能过渡到什么阵容；不要默认主C，不要强推追3，装备只说“可携带”。

Failure Type:

Prompt / Query

Decision:

已新增 `role_query=work`，并在 prompt 中把打工问题和主C问题分开。

Status:

Fixed

### Case 2

Question:

`二阶段几费卡打工强？`

Result:

之前回答会说当前知识库没有 1/2 费打工数据，并引用不足以支撑的商店概率判断。

Score: 5/10

Expected:

承认当前没有专门的二阶段打工评分；如果有 1/2 费英雄画像，可以给“可参考的低费过渡卡”，但不能把中后期强度当成二阶段打工强度。

Failure Type:

Query / Data

Decision:

已新增 `work_score`（打工评分）和 `work_reason`（打工原因），用英雄费用、阶段、阵容承接和装备承载数据综合排序。注意它仍是结构化启发式评分，不等于真实对局胜率。

Status:

Fixed

### Case 3

Question:

`炸弹人能玩吗？`

Result:

之前会回答当前知识库没有命中。

Score: 4/10

Expected:

如果当前赛季有吉格斯，应按“炸弹人=吉格斯”理解；如果当前赛季没有吉格斯，应明确说“已识别炸弹人=吉格斯，但当前知识库没有该英雄”。

Failure Type:

Alias / Data

Decision:

`aliases.json` 已加入 `炸弹人 -> 吉格斯`，后续需要用当前 Set 数据验证是否有该英雄。

Status:

Partially Fixed
